## Déclaration de Sécurité en Cybersécurité
**Version :** 1.0  
**Statut :** Actif (Aligné sur ISO 27001)

## 1. Sécurité de l'Architecture Système
Notre portail de documentation suit la philosophie **JAMstack**, réduisant considérablement la surface d'attaque par rapport aux plateformes CMS traditionnelles basées sur des bases de données.

*   **Génération Statique :** L'interface utilisateur est pré-rendue en HTML statique. Il n'y a pas de serveur d'applications en direct ni de base de données SQL exposés à l'internet public, ce qui atténue les risques d'injection SQL (**SQL Injection**) et de script intersites (**Cross-Site Scripting (XSS)**).
*   **Déploiement en Bord de Réseau (Edge) :** Les actifs sont servis via un Réseau de Diffusion de Contenu (CDN) mondial avec une protection DDoS intégrée et un chiffrement SSL/TLS en transit.

## 2. IA et Confidentialité des Données (Données en Transit)
Pour alimenter nos fonctionnalités de recherche sémantique et de traduction automatique, nous utilisons des API d'IA externes (Google AI Studio et Pinecone).

*   **Minimisation des Données :** Aucune information personnellement identifiable (**PII**) ni aucune information d'identification interne propriétaire ne sont jamais incluses dans les fichiers source Markdown ni envoyées aux points d'accès de traitement d'IA.
*   **Sécurité du Transport :** Toutes les communications avec Google AI Studio et Pinecone sont effectuées via des connexions chiffrées **HTTPS (TLS 1.2+)**.
*   **Sécurité de la Base de Données Vectorielle :** Notre index Pinecone est protégé par une authentification basée sur une clé API. Les métadonnées stockées dans la base de données vectorielle sont limitées aux extraits de documentation et aux URL accessibles au public.

## 3. Gestion des Secrets et Sécurité CI/CD
Conformément aux meilleures pratiques **ISO 27001**, nous traitons notre pipeline d'automatisation comme un environnement sécurisé.

*   **Secrets GitHub :** Les identifiants sensibles, y compris `PINECONE_API_KEY`, `HUGGINGFACE_API_KEY` et `GOOGLE_API_KEY`, ne sont **jamais** codés en dur dans le code source. Ils sont stockés dans le coffre-fort de secrets chiffré de GitHub.
*   **Moindre Privilège :** Les actions GitHub sont configurées avec les permissions minimales requises (`contents: write` uniquement pour les commits automatisés).
*   **Pistes d'Audit :** Chaque modification apportée à la documentation ou au pipeline d'IA est gérée par version via Git, fournissant une piste d'audit permanente.

## 4. Contrôle d'Accès (IAM)
*   **Accès Administratif :** L'accès au dépôt de production et aux tableaux de bord d'API est restreint via l'authentification multifacteur (MFA).
*   **Protection du Déploiement :** Les poussées directes vers la branche `main` sont restreintes via des règles de protection de branche, nécessitant des révisions de demandes de tirage (pull requests) pour toutes les mises à jour de la documentation.

## 5. Gestion des Vulnérabilités
*   **Analyse des Dépendances :** Nous utilisons des outils automatisés (par exemple, GitHub Dependabot) pour surveiller et corriger les vulnérabilités dans nos dépendances Python et Node.js.
*   **Analyse Statique :** Les scripts `ingest.py` et `re_embed.py` sont régulièrement audités pour détecter les pratiques de codage non sécurisées.
