## Ce que vous allez apprendre

Dans ce guide, vous allez :

* authentifier votre première requête API avec OAuth 2.0
* comprendre comment l’accès à l’API est contrôlé
* voir comment l’authentification est liée au suivi d’usage et à la facturation

→ Voir : [Guide de monétisation API](/fr/api-monetization/api-monetization-guide)

---

## Avant de commencer

Assurez-vous d’avoir :

* une clé API ou des identifiants client
* un accès au tableau de bord API
* un outil comme curl ou Postman

---

## Étape 1 : Obtenir vos identifiants API

Depuis votre tableau de bord, créez :

* Client ID
* Client Secret

Ces identifiants permettent d’identifier votre application et de contrôler l’accès.

L’accès est souvent lié à un **plan ou abonnement**.

→ Voir : [Gestion des droits (Entitlements)](/fr/api-monetization/entitlement-management-api)

---

## Étape 2 : Demander un jeton d’accès

Utilisez OAuth 2.0 pour obtenir un jeton :

```bash id="f3l2xv"
curl -X POST https://api.votreservice.com/oauth/token \
  -d "client_id=YOUR_CLIENT_ID" \
  -d "client_secret=YOUR_CLIENT_SECRET" \
  -d "grant_type=client_credentials"
```

### Réponse

```json id="n7m4yb"
{
  "access_token": "YOUR_ACCESS_TOKEN",
  "expires_in": 3600
}
```

---

## Étape 3 : Faire votre première requête API

Utilisez le jeton :

```bash id="xq9s2r"
curl https://api.votreservice.com/v1/resource \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

---

## Étape 4 : Ce qui se passe en arrière-plan

À chaque requête :

1. votre identité est authentifiée
2. vos droits d’accès sont vérifiés (plan, permissions)
3. votre usage est enregistré pour le suivi et la facturation

→ Voir : [Architecture de suivi d’usage](/fr/api-monetization/usage-tracking-architecture)
→ Voir : [Facturation basée sur l’usage](/fr/api-monetization/usage-based-billing)

---

## Usage et limites

Votre utilisation de l’API peut être limitée selon votre plan :

* nombre de requêtes par minute
* nombre de requêtes par mois
* accès à certains endpoints

En cas de dépassement :

* erreurs possibles
* ou facturation supplémentaire

→ Voir : [Gestion des droits (Entitlements)](/fr/api-monetization/entitlement-management-api)

---

## Erreurs courantes

### 401 Non autorisé

* jeton d'accès invalide ou expiré

### 403 Interdit

* accès non autorisé pour votre plan

### 429 Trop de requêtes

* limite de taux dépassée

---

## Bonnes pratiques

* Stockez vos identifiants de manière sécurisée.
* Renouvelez vos jetons d'accès avant expiration.
* Surveillez votre usage pour éviter les limites.
* Optimisez vos requêtes pour réduire les coûts.

→ Voir : [Architecture de suivi d’usage](/fr/api-monetization/usage-tracking-architecture)

## FAQ

### Qu’est-ce que OAuth 2.0 ?

OAuth 2.0 est un protocole standard utilisé pour sécuriser l’authentification des requêtes API.

---

### Pourquoi ai-je besoin d’un jeton d’accès ?

Le jeton d'accès prouve votre identité et permet à l’API de contrôler et suivre votre usage.

---

### Chaque requête est-elle comptabilisée ?

Oui. Chaque requête authentifiée est généralement suivie et peut être facturée selon votre plan.

---

### Que se passe-t-il si je dépasse mes limites ?

Vos requêtes peuvent être bloquées ou facturées selon votre modèle tarifaire.
